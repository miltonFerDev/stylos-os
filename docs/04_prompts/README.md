# Prompts

## Propósito

Almacenar y versionar los prompts usados para análisis con IA. Cada prompt debe incluir su contexto, objetivo y ejemplos de uso.

## Convenciones

- Cada prompt se documenta en un archivo Markdown separado dentro de esta carpeta.
- Un prompt debe incluir: contexto requerido, instrucción principal, formato de salida esperado y ejemplos.
- Los prompts se mejoran iterativamente. Cuando un prompt cambia significativamente, se guarda la versión anterior como referencia.
- La IA siempre debe recibir contexto real del negocio (datos, criterios, eventos) y no trabajar con información suelta.

## Estructura de un prompt

```markdown
# [Nombre del prompt]

## Objetivo
[Qué busca lograr]

## Contexto requerido
[Qué datos o documentos necesita la IA]

## Instrucción
[El prompt propiamente dicho]

## Formato de salida
[Cómo se espera la respuesta]

## Ejemplo de uso
[Cómo se usaría en una consulta real]

## Historial de versiones
[Cambios relevantes]
```
